## 1. 设计理念

### 1.1 核心问题

传统论文研究流程存在三大痛点：
- **发现低效**：手动搜索ArXiv，每天100+论文难以筛选
- **理解困难**：阅读英文论文耗时，难以快速把握核心
- **复现困难**：代码缺失、环境配置复杂、实验难以验证

### 1.2 设计目标

构建一个**端到端的AI论文研究自动化平台**：

```
用户查询 → 智能搜索 → 多级过滤 → 深度分析 → 实验复现 → 知识积累
```

### 1.3 核心理念

| 理念 | 说明 |
|------|------|
| **自动化** | 每日自动爬取、分析、报告 |
| **智能化** | AI摘要、智能评分、趋势预警 |
| **可扩展** | 模块化架构，易于添加新功能 |
| **生产级** | 容器化部署、健康检查、自愈机制 |

## 3. 核心算法

### 3.1 多级过滤漏斗

**目标**：从100+篇论文中筛选出最有价值的20篇

```
Stage 1: TF-IDF快速过滤
  - 领域关键词匹配
  - 重要词汇匹配
  - 阈值: 0.2
  - 效果: 过滤60%无关噪音
  
Stage 2: LLM深度评分
  - 启发式评分算法
  - 考虑新颖性、性能指标、引用数
  - 输出: 1-10分评分
  - 效果: 精准筛选Top 20
```

### 3.2 AI摘要生成

**结构化输出模板**：
```json
{
  "core_problem": "≤25字，核心问题",
  "innovation": "≤50字，创新方法",
  "conclusion": "含量化结果",
  "limitation": "≤40字，改进方向"
}
```

**防滥用机制**：
- 摘要截断至500字符
- 缓存结果避免重复调用
- 失败时返回默认摘要

### 3.3 趋势预警算法

```python
# 计算增长率
growth_rate = (recent_7d - previous_7d) / previous_7d * 100

# 计算波动系数
volatility = std(daily_counts) / mean(daily_counts)

# 触发条件
if growth_rate > 35% and volatility > 0.4:
    alert_level = "HIGH"
```

## 5. 部署架构

### 5.1 Docker Compose

```yaml
services:
  app:        # 主应用（爬取+分析）
  web:        # Web API（端口8000）
  scheduler:  # 定时任务（每日8:00）
```

### 5.2 健康检查

- Web服务: 30秒检查一次 http://localhost:8000/api/health
- Scheduler: 检查crond进程和配置文件
- 自动重启: 连续失败3次后自动重启

### 5.3 自愈机制

- monitor.sh: 每分钟检查容器状态
- 自动重启异常容器
- 日志记录所有操作
