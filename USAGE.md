## 2. 核心功能演示

### 2.1 使用 /docs 交互式文档

这是最直观的测试方式：

```
步骤：
1. 浏览器打开 http://localhost:8000/docs
2. 看到 Swagger UI 界面，列出所有 API 端点
3. 点击任一端点（如 GET /api/papers）
4. 点击 "Try it out" 按钮
5. 填写参数（如 limit=5, topic=AI_Agent）
6. 点击 "Execute" 执行请求
7. 下方 Response 区域显示返回的 JSON 数据
```

### 2.2 搜索论文

**方式一：在 /docs 页面操作**

```
1. 找到 GET /api/papers/search 端点
2. 点击展开 → 点击 "Try it out"
3. 在 q 参数输入: AI
4. 在 topic 参数输入: AI_Agent（可选）
5. 点击 Execute
6. 查看返回的论文列表
```

**方式二：命令行**

```bash
# 搜索包含"AI"的论文（返回前5篇）
curl "http://localhost:8000/api/papers/search?q=AI&limit=5"

# 按领域搜索
curl "http://localhost:8000/api/papers/search?q=agent&topic=AI_Agent"

# 搜索高分论文（评分≥7）
curl "http://localhost:8000/api/papers/search?q=LLM&min_score=7"
```

### 2.3 查看论文详情

```bash
# 获取单篇论文完整信息
curl "http://localhost:8000/api/paper/2607.18006v1"

# 返回内容包括：
# - id, title, abstract, authors
# - llm_score (0-10评分)
# - llm_tags (标签)
# - analysis (深度分析: 问题/方法/优点/改进建议)
```

### 2.4 对比论文

```bash
curl "http://localhost:8000/api/papers/compare?id1=2607.18006v1&id2=2607.18080v1"
```

## 4. 定时任务说明

### 4.1 Scheduler 服务

`paper-pipeline-scheduler` 容器自动执行论文爬取管道。

**执行计划**：每天 08:00 自动运行

**工作流程**：
```
08:00 触发 → 搜索最新论文 → 分析评分 → 生成报告 → 发送邮件
```

### 4.2 查看日志

```bash
# 查看容器运行状态
docker compose ps

# 实时查看日志
docker compose logs -f scheduler

# 查看任务执行日志
docker compose exec scheduler cat /app/data/cron.log
```

### 4.3 手动触发

```bash
# 手动运行一次
docker compose exec app python scripts/run_pipeline.py --mode daily

# 查看最新报告
docker compose exec app ls -la reports/
```

### 4.4 修改执行时间

编辑 `docker-compose.yml` 中 scheduler 服务的 command：

```yaml
# 原配置：每天 08:00
command: >
  sh -c "echo '0 8 * * * ...' > /etc/crontabs/root && crond -f"

# 改为每天 07:30
command: >
  sh -c "echo '30 7 * * * ...' > /etc/crontabs/root && crond -f"
```

## 5. 常见问题

| 问题 | 解决方案 |
|------|---------|
| API返回空数据 | `docker compose cp data/ web:/app/data/` |
| 服务无法访问 | `docker compose restart web` |
| 端口被占用 | `netstat -tlnp \| grep 8000` |
| 更新代码后不生效 | `docker compose up -d --build` |
| 停止所有服务 | `docker compose down` |

      "command": "python3",
