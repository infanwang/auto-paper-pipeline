# 📖 Auto Paper Pipeline 使用指南

## 快速开始

### 1. 启动服务

```bash
cd auto-paper-pipeline
docker compose up -d
```

### 2. 访问Web界面

打开浏览器访问：**http://localhost:8000**

---

## 🔍 搜索论文

### 方法1：API直接搜索

```bash
# 搜索关键词
curl "http://localhost:8000/api/papers/search?q=agent&limit=5"

# 按领域筛选
curl "http://localhost:8000/api/papers/search?q=LLM&topic=AI_Agent"
```

### 方法2：Python调用

```python
import requests

# 搜索论文
response = requests.get("http://localhost:8000/api/papers/search", params={
    "q": "LLM inference",
    "limit": 10
})
papers = response.json()["papers"]

for p in papers:
    print(f"{p['title'][:50]} | Score: {p['score']}")
```

## 📄 论文详情

### 获取单篇论文详情

```bash
curl "http://localhost:8000/api/paper/2607.16193"
```

返回信息包括：
- 标题、摘要、作者
- 评分 (0-10)
- 标签
- 深度分析（问题、方法、优点、改进建议）

## 🔌 MCP接入（Claude/Cursor）

在 Claude Desktop 配置中添加：

```json
{
  "mcpServers": {
    "paper-pipeline": {
      "command": "python",
      "args": ["/path/to/src/mcp_server/server.py"]
    }
  }
}
```

然后在Claude中直接说：
- "搜索LLM agent相关论文"
- "查看2607.16193的详细信息"
- "各领域有多少论文？"
- "评分最高的5篇论文"

## 🐛 常见问题

### Q: API返回空数据？
A: 检查数据文件是否存在：
```bash
docker compose exec web ls /app/data/
```

### Q: 服务无法访问？
A: 检查端口是否被占用：
```bash
docker compose ps
curl http://localhost:8000/health
```

### Q: 搜索无结果？
A: 尝试更宽泛的关键词，如 "LLM" 而不是 "LLM inference optimization"
